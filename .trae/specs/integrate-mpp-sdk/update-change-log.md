## 具体变更内容

### 1. spec.md 更新

**新增内容：**
- 添加了更详细的 `Why` 说明
- 扩展了 `What Changes` 列表，明确所有变更
- 添加了多个需求场景：
  - MPP SDK 依赖获取需求
  - MPP SDK 集成需求（包含 3 个场景）
  - 持久化存储需求（包含 2 个场景）
  - ESM 模块兼容性需求
- 更新了 MODIFIED Requirements，明确公共接口哪些改变哪些保持
- 添加了 REMOVED Requirements，说明移除的模拟实现

**影响范围：** 仅规格文档，不影响代码

---

### 2. tasks.md 更新

**结构变更：**
- 原：5 个顶级任务 → 新：8 个顶级任务
- 原：无子任务分解 → 新：每个顶级任务分解为具体子任务
- 添加了 Acceptance Criteria 每个任务
- 保持了任务依赖关系图
- 根据实际进度标记了已完成任务 (Task 1-2 已完成)

**新增任务：**
- **Task 5**: ESM 模块兼容性验证 - 新增，确保兼容项目整体 ESM
- **Task 6**: 依赖兼容性与类型检查 - 新增，专门验证 viem 和 mppx 版本
- **Task 8**: 全面功能验证与冲突检查 - 新增，包含 ESLint Prettier 检查

**已完成任务（保持不变）：**
- ✅ Task 1: 定位 MPP SDK 依赖
- ✅ Task 2: 安装依赖并解决冲突

**待完成任务：**

- ⬜ Task 4: 验证上层调用适配
- ⬜ Task 5: ESM 模块兼容性验证
- ⬜ Task 6: 依赖兼容性与类型检查
- ⬜ Task 7: 构建验证与功能测试
- ⬜ Task 8: 全面功能验证与冲突检查
